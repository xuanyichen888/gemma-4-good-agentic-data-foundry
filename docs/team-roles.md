# Team Roles

## Technical lead

Owner: you

Responsibilities:

- repo architecture
- Gemma 4 integration
- database construction pipeline
- schema inference and validation
- natural-language to SQL workflow
- evaluation table and technical writeup

Recommended weekly output:

- short demo every 2-3 days
- architecture diagram
- benchmark/evaluation table
- final GitHub README

## Business teammate 1

Suggested role: user and impact lead

Responsibilities:

- define target organization persona
- collect examples of messy intake/reporting workflows
- build impact metrics
- write user story
- help justify why local-first matters

Concrete tasks:

- interview 2-3 potential users or simulate interviews from public workflows
- create before/after workflow diagram
- define time-saved and cost-saved assumptions
- prepare impact section of writeup

## Business teammate 2

Suggested role: pitch and demo lead

Responsibilities:

- video script
- slide/storyboard
- Kaggle writeup polish
- demo narrative
- visual assets

Concrete tasks:

- draft 3-minute video script
- prepare a one-page pitch
- create demo persona and scenario
- coordinate final recording

## Team rule

Every feature must answer one of these questions:

- Does this help the target user complete the workflow?
- Does this make the system more trustworthy?
- Does this make the Gemma 4 usage more meaningful?
- Does this improve the submission story?
